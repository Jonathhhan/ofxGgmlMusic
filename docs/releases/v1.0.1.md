# ofxGgmlMusic v1.0.1

## Compatibility

- openFrameworks: `of_v20260406_vs_64_release`
- minimum `ofxGgmlCore`: `1.0.1` / commit `2171e8b`
- `OFXGGML_MUSIC_VERSION_STRING`: `1.0.1`

## Highlights

- Added independent Music addon version metadata.
- Exposed version metadata through `ofxGgmlMusic.h`.
- Kept `ofxGgmlMusic` separate from `ofxGgmlAudio`.
- Documented the release checklist and release policy for music workflows.
- Preserved procedural prompt-to-music generation as the first testable
  generation backend.

## Runtime Scope

This release includes music request/result types, generation request/result
types, generation backend interfaces, a deterministic procedural WAV backend,
manifest/history helpers, MIDI sidecars, stem exports, root-level analysis and
generation examples, and a no-IDE procedural generation CLI.

It does not include a model-backed MusicGen, diffusion, SampleRNN, or external
bridge runtime yet. Those remain roadmap tasks.
